# Source acquisition and rebuild notes

This project currently rebuilds the static Tide Atlas from archived local inputs plus one reproducible public map source. The notes below document the current provenance and the commands needed to regenerate the committed derived data files.

## Tide-gauge atlas inputs

The sea-level station metadata and annual station series are archived inside this repository from an earlier University of Hawaii Sea Level Center fast-delivery tide-gauge workflow.

Current local inputs:

- `Old Junk/Sea_Level_Map_2/data/data.json` — station metadata used by `scripts/build-data.mjs`, including station file names, display names, country labels, coordinates, and source year ranges.
- `Old Junk/Sea_Level_Data_Groomer/data/` — per-station CSV files consumed by `scripts/build-data.mjs`; each row includes a year and a water-level value from the archived groomed UH input set.

The current builder intentionally treats these archived files as canonical local source inputs. To rebuild the generated atlas JSON after changing either input set, run:

```bash
npm run build:data
npm run validate:data
```

This writes `static/data/sea-level-atlas.json`. The builder aggregates station rows into annual means, filters sparse years with fewer than 90 observations, computes each station's first-five-valid-years anomaly baseline, calculates ordinary least-squares trend values in millimeters per year, and writes deterministic metadata derived from the data year range.

## World land polygons

The map land layer is generated from the `world-atlas@2` package's hosted TopoJSON file:

- Source URL: `https://cdn.jsdelivr.net/npm/world-atlas@2/land-110m.json`
- Upstream source credit: `world-atlas@2 land-110m, derived from Natural Earth 1:110m land polygons`
- Rebuild script: `scripts/build-world-map.mjs`

To refresh the simplified land polygons, run:

```bash
npm run build:world-map
npm run validate:export
```

This downloads the TopoJSON source, decodes land arcs, and writes `static/data/world-land.json` for the browser map renderer. The static export validator then confirms the generated `public/data/world-land.json` exists after a Hugo build.

## Reproducibility checklist

When source data changes, run the full local validation path before committing generated files:

```bash
npm test
npm run build:data
npm run validate:data
npm run build:world-map
npm run validate:export
```

Expected generated outputs:

- `static/data/sea-level-atlas.json`
- `static/data/world-land.json`
- `public/` files regenerated by `npm run validate:export` / Hugo

## Remaining provenance gap

The repository documents how to rebuild from the archived UH-derived local files. A future improvement is to add a scripted fetch/import path for the original University of Hawaii Sea Level Center source files if stable public endpoints for the exact archived inputs are identified.
